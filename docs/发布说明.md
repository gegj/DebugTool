# 发布说明

## 发布步骤

1. 确认 `csharp/Program.cs` 中版本号已更新：

```csharp
[assembly: AssemblyVersion("1.1.7.0")]
[assembly: AssemblyFileVersion("1.1.7.0")]
[assembly: AssemblyInformationalVersion("1.1.7")]
private const string AppVersion = "1.1.7";
```

2. 提交代码并推送到 GitHub。

3. 推送版本 tag 后，由 GitHub Actions 自动构建并发布 Release。

## 发布文件

Release 只需要包含：

```text
DebugTool.exe
```

`logo.ico` 已编入 EXE，运行时不需要放在同目录。

当前已编译下载地址：

```text
https://github.com/gegj/DebugTool/releases/latest/download/DebugTool.exe
```

## 自动发布流程

仓库已配置 GitHub Actions。推荐只在推送版本 tag 时发布新版。

平时开发：

```bat
git add .
git commit -m "更新功能"
git push
```

需要发布新版时：

```bat
git tag v1.1.7
git push origin v1.1.7
```

打 tag 前必须先把 `csharp/Program.cs` 里的版本号改成同一个版本：

```csharp
[assembly: AssemblyVersion("1.1.7.0")]
[assembly: AssemblyFileVersion("1.1.7.0")]
[assembly: AssemblyInformationalVersion("1.1.7")]
private const string AppVersion = "1.1.7";
```

GitHub Actions 会自动：

- 使用 Windows runner 编译 `DebugTool.exe`
- 根据 tag 生成 `latest.json`
- 创建 GitHub Release
- 上传 `DebugTool.exe` 和 `latest.json`

## 更新 JSON

发布新版本时，同步更新 `latest.json` 并提交到仓库：

```json
{
  "version": "1.1.7",
  "url": "https://github.com/gegj/DebugTool/releases/latest/download/DebugTool.exe",
  "notes": "更新说明"
}
```

注意：`version` 必须大于旧版程序内置的 `AppVersion`，否则旧版不会提示更新。

## 远程地址

程序内置检测地址：

```text
https://github.com/gegj/DebugTool/releases/latest/download/latest.json
```

GitHub Actions 发布时会自动上传：

```text
DebugTool.exe
latest.json
```

## 版本检查

如需检查版本，可下载 Release 中的 `DebugTool.exe` 后用 PowerShell 查看：

```powershell
(Get-Item .\DebugTool.exe).VersionInfo
```

重点确认：

```text
FileVersion
ProductVersion
```
